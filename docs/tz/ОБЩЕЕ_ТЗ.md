## Как подключить экран к навигации

Все экраны — заглушки `PlaceholderScreen`. Ваша задача — заменить их на реальные экраны.

**Файл:** `MainScreen.kt` → функция `RoleScreenRouter()`

```kotlin
// БЫЛО:
route == "athlete/dashboard" -> PlaceholderScreen("Главная спортсмена", user, onSignOut)

// СТАЛО:
route == "athlete/dashboard" -> AthleteDashboardScreen(user = user)
```

## Отступы

- Между экраном и контентом: `padding(horizontal = 20.dp)`
- Сверху: `Spacer(Modifier.height(16.dp))` после statusBarsPadding
- Между секциями: `Spacer(Modifier.height(28.dp))`
- Между карточками: `Arrangement.spacedBy(8-10.dp)` или `Spacer(8.dp)`
- Между заголовком секции и контентом: `Spacer(Modifier.height(12.dp))`
- Внутри карточки: `padding(14.dp)` или `padding(20.dp)`
- Снизу экрана: `Spacer(Modifier.height(32.dp))`

- ✅ Создавать моковые данные: `data/mock/{ВашаРоль}MockData.kt`

- ✅ Импортировать палитру из `DarkTheme` (`import com.ileader.app.ui.components.DarkTheme`) и использовать через `DarkTheme.Bg`, `DarkTheme.CardBg` и т.д. Можно создавать локальные alias: `private val Bg = DarkTheme.Bg`

- ✅ Использовать `verticalScroll(rememberScrollState())` для длинных экранов
- ✅ Использовать `LazyColumn` для списков с однотипными элементами
- ✅ Добавлять анимации: FadeIn (см. выше), `animateContentSize`, `animateFloatAsState`
